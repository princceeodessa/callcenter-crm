# План распознавания эскизов замерщика

## Цель

После загрузки фото чертежа получить черновую геометрию комнаты, длины стен в сантиметрах и контрольные значения площади/периметра, чтобы оператор или технолог могли быстро поправить результат в проектировке.

## Почему это реально

Типовой эскиз замерщика содержит сразу несколько источников данных:

- контур комнаты, обычно нарисованный ручкой;
- длины стен, чаще всего в сантиметрах;
- контрольные подписи `S` и `P`;
- иногда высоту, нишу, карниз, трубы и комментарии.

На примере вида `172`, `251`, `S 4,32`, `P 8,46` можно:

- распознать прямоугольную комнату;
- перевести длины в `172 см` и `251 см`;
- проверить, что площадь близка к `4.32 м2`;
- проверить, что периметр близок к `8.46 м`.

Это повышает надежность по сравнению с распознаванием одних только линий.

## Рекомендуемый pipeline

1. Предобработка изображения.
   - выровнять фото;
   - усилить контраст;
   - убрать фон и шум;
   - выделить рукописные линии.

2. OCR по подписям.
   - считать числа рядом со стенами;
   - найти `S`, `P`, `H`, подписи ниш/карнизов;
   - нормализовать десятичные разделители.

3. Детектор контура.
   - найти внешний полигон комнаты;
   - выделить основные вертикальные/горизонтальные сегменты;
   - для простых эскизов строить сразу прямоугольник/трапецию.

4. Сопоставление чисел со стенами.
   - привязать распознанные длины к ближайшему сегменту;
   - если сегментов мало и чисел мало, использовать простую эвристику;
   - если есть конфликт, снижать confidence.

5. Валидация геометрии.
   - сверить площадь и периметр с `S` и `P`, если они указаны;
   - если отклонение маленькое, повышать confidence;
   - если отклонение большое, помечать как черновик для ручной правки.

6. Построение чернового проекта.
   - создать комнату;
   - записать точки контура;
   - заполнить длины стен;
   - показать оператору результат поверх фото.

## Что делать в первой версии

Не пытаться сразу распознавать все сложные комнаты.

Первая версия должна уметь:

- прямоугольник;
- Г-образную комнату;
- чтение длин стен в сантиметрах;
- чтение `S` и `P`;
- построение черновика с ручной доработкой.

## Что важно для UX

- результат не должен сохраняться молча;
- оператор должен видеть: найденные стены, длины, площадь, периметр и confidence;
- должен быть режим `Применить как черновик`;
- исходное фото должно оставаться подложкой для ручной правки.

## Что хранить в системе

- исходное фото;
- сырой OCR-текст;
- распознанные длины стен;
- распознанный контур;
- контрольные `S` и `P`;
- confidence по всему распознаванию и по каждому сегменту.

## Практичный стек для старта

- OCR: сначала обычный OCR + дополнительная нормализация цифр;
- контур: OpenCV или аналогичная обработка изображения;
- логика сборки комнаты: отдельный Laravel-сервис;
- итог: черновой JSON с точками и длинами для проектировки.
